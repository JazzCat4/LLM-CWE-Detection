3. Dependency Analysis Graph
    Reachability:
        - secret_data reached acc_reg (secret_data -> update_logic -> acc_next -> acc_reg -> output_acc)
        - secret_bit reaches acc_reg through control flow (secret_bit -> update_logic -> acc_next -> acc_reg)
        - rst_n directly reaches acc_reg (rst_n -> reset_logic -> acc_reg)
        - clk reachces acc_reg through state update trigger.
        - Every input can reach acc_reg, which can reach the output.
    Dominance:
        - rst_n dominates acc_reg.
        - clk dominates all state transitions.
        - secret_bit dominates the use of secret_data
        - acc_reg domaintes the output acc.
    Taint Analysis:
        - secret_data leads to full taint propagation into persistent state and output.
        - secret_bit has a control-flow taint. The output reveals whether secret-dependent updates occurred.
        - acc_reg becomes a derived secret asset because it contains accumulated secret-dependent values.
        - If output_acc is observable outside the secure domain, both data-taint and control-taint leak.
