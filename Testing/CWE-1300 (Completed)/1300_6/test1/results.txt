1. Type Classification
    CWE-226 and CWE-1300
2. Asset Identification
    Assets
        - Secret Bit (secret_bit)
        - Accumulator register (acc)
        - Input Data (data_in)
    Privilege Domain Boundaries
        - Secret -> datapath control boundary (secret_bit drives the ternary operator between acc + data_in or acc - data_in)
        - On-chip domain assumption
        - No explicit secure/non-secure partitioning.
    Clock Behavior
        - Positive Edge truggered sequential logic
        - Single clock domain, no clock gating/division
    Reset Schemes
        - Asynchronous single active low reset, global set
    Adversary Capabilities
        - Manipating functional inputs data_in. (can choose values to leak data)
        - Observating outputs (can read acc if exposed to another module or off-chip)
        - Clock/reset maniplation
        - No explicit JTAD/debug pins.
    Present Security Rules
        - Reset rule (acc becomes 0 on !rst_n)
    Absent Security Rules
        - No restriction on secret_bit usage
        - No access control on acc
        - No domain separation
        - No side-channel hardening
    FSM
        - No explicit state
        - State variable acc
        - State domain: All 2^16 values of acc.
        - On the rising edge of clk with rst_n == 1, acc_next becomes either one above or one below acc_reg depending on the value of secret_bit.
        - On !rst_n, acc_next becomes 0.
        - No defined illegal states.
        - branching on secret_bit is the key transition rule.
3. Dependency Analysis Graph
    Reachability:
        - Every input and its prior state are reachable to the next state.
        - rst_n influences all future acc values
    Dominance:
        - clk dominates all state updates
        - Reset assignment is always chosen over going to the next acc.
        - Secret_bit dominates over branch choice
    Taint Analysis:
        - acc becomes fully tainted by data_in over time. (data_in -> add/sub branch -> branch selection -> next acc)
        - Taint flows via control dependency (secret_bit -> branch selection -> next acc)
        - acc is jointly tainted by data_in (data taint) and secret_bit (control taint).
4. CWE-Driven Review
    CWE-226:
        - No explicit/guaranteed scrub of secret-derived state before reuse/exposure
        - reset coverage depends on external rst_n behavior
    CWE-1300:
        - Secret-dependent operand selection leaks through side channels.
        - Secret-tained state evolution of acc
