        - Secret -> computation boundary

        - Single synchronous clock that drives all sequential logic
        - One bit is process per rising edge while bit_idx < 8.
    Reset Schemes
        - Asynchronous active-low reset
        - reset effects bit_idx (0), match_bit (1), access_granted (0)
    Adversary Capabilities
        - Control of key input key_in
        - Control of command input cmd_enter
        - Clock/reset manipulation
        - Observation of authorization output
    Present Security Rules
        - Grants access only if all 8 bits of key_in match correct_pin.
        - Compare bits sequentially from 0 to 7 while bit_idx < 8.
        - On global reset, clear authorization and reinitialize comparison state.
        - On cmd_enter, restart comparison.
    FSM (Implicit)
        - S0: Reset/Idle State (S0 -> S1)
        - S1: Ready For Command State (S1 -> S2)
        - S2: Comparison in Progress (S2 -> S3)
        - S3: Decision and Output (S3 -> S1)
        - Stale authorization state, where access_granted can stay 1 until S3 is reached again.
        - Glitch induced partial reset
        - Non-monotonic bit_idx behavior under faults.

    CWE-1256:

    CWE-1262:

        - Privilege/trust not hardware-derived.
        - Authorization output not protected
