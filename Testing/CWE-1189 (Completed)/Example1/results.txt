TYPE CLASSIFICATION
Memory Controller / DMA, SoC bus/ interconnect / MPU, Clock / reset logic
Relevant CWEs: 226, 1189, 1260, 1262, 1247, 1234

ASSET IDENTIFICATION
Assets: shared memory contents, CPU and Peripheral data paths, turn bit
Priv. domain: Secure CPU vs untrusted peripheral, read vs write privileges
Clock and reset: Turn updated on posedge clk, memory is not reset or scrubbed
Adversary capabilities: Peripheral as adversary, potential manipulation of clock/reset
Security rules: Weak access rule utilizing turn. No Address-range restrictions, priv. bits, lifecycle checks, write protection, read protection, or scrubbing on reset.
States: Turn 0 = CPU, Turn 1 = Peripheral, transition between each every cycle

PDG ANALYSIS
Reachability: Any data written by the secure CPU is reachable by the untrusted peripheral through the shared memory
Dominance: The arbitration bit turn is a dominant control point for all accesses, but it does not enforce any security separation-only temporal scheduling.
Taint: Taint from the untrusted peripheral directly contaminates the shared memory and can be observed by the secure CPU. Sensitive data from the secure CPU flows back to the untrusted peripheral via the same memory. There is no taint containment.
